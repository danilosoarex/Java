Service Resgistry -> É basicamente uma agenda telefônica dos microsserviços.
  Um servidor central guarda:
    - nome do serviço
    - endereço (IP)
    - porta
    - status (vivo ou morto)
  Quando o serviço cai → ele é removido automaticamente.

  Exemplo famoso em Java (Spring Cloud)
  Eureka Server -> Ele é literalmente o “cartório” dos microsserviços.

Service Discovery -> É o mecanismo de localização dinâmica de serviços em tempo de execução.
  Um cliente inteligente que resolve nomes lógicos de serviços em endereços reais e escolhe dinamicamente para quem enviar a requisição.
  Ele é basicamente: DNS + Load Balancer + Health Check + Retry automático, tudo embutido na chamada HTTP.

  Tipos de Service Discovery
    Client-side discovery (mais comum no Spring) -> O próprio microsserviço resolve quem chamar.
    pedido-service → descobre → usuario-service
      Exemplo: 
        Spring Cloud LoadBalancer
        OpenFeign
    
    Server-side discovery (Kubernetes) -> Um proxy resolve por você.
    pedido-service → proxy → usuario-service
      Exemplo: 
        Kubernetes Service
        API Gateway
        Nginx / Envoy

   Analogia:
   Service Registry → lista de motoristas disponíveis
   Service Discovery → o algoritmo escolhendo QUAL motorista vai te buscar agora

   Service Registry armazena informações.
   Service Discovery toma decisões em tempo real.
